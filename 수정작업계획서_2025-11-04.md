# 수정 작업 계획서 - 테스트 이슈 분석 및 개선 방안

## 문서 개요

본 문서는 테스트 결과를 바탕으로 URL/테스트 화면별로 분류하여 체계적인 수정 작업 계획을 수립합니다.

**작성일**: 2025년 11월 4일  
**대상 환경**: https://app-was-int-dev.azurewebsites.net/

---

## 요구사항 및 현황 분석

### 전체 이슈 현황
- **총 이슈 수**: 약 30개 이상
- **우선순위 높음**: 18개
- **우선순위 중간**: 12개
- **우선순위 낮음**: 5개

### 주요 문제 유형 분류
1. **데이터베이스 관련** (외래키 제약, 사용자 데이터 부재): 8건
2. **API 에러** (404, 500, 연결 실패): 10건
3. **UI/UX 문제** (드래그/선택 불가, 검색 미작동): 6건
4. **워크플로우 실행 로직** (노드 타입 미지원, 변수 참조 오류): 5건
5. **상태 관리** (타이밍 불일치, UI 갱신 안됨): 4건

## 핵심 개선 포인트 도출

### 1. 데이터베이스 무결성 문제 (최우선)
- **문제**: Dictionary, 테마 클러스터, 개인화 관련 화면에서 외래키 제약 조건 위반으로 인한 데이터 저장 실패
- **영향도**: 높음 - 사용자가 데이터를 저장할 수 없어 핵심 기능이 동작하지 않음
- **근본 원인**: 외래키 참조 무결성 체크 없이 데이터 저장 시도

### 2. 사용자 데이터 부재 문제
- **문제**: 개인화 관련 화면에서 사용자 프로필/위험 프로필이 존재하지 않아 404 에러 발생
- **영향도**: 높음 - 개인화 기능 전체가 동작하지 않음
- **근본 원인**: 사용자 생성 시 기본 프로필 데이터 생성 로직 부재

### 3. 워크플로우 실행 엔진 노드 타입 미지원
- **문제**: `theme_classifier`, `alert` 노드 타입이 실행 엔진에서 지원되지 않음
- **영향도**: 높음 - 워크플로우 실행 실패
- **근본 원인**: 프론트엔드 타입 정의와 백엔드 실행 로직 불일치

### 4. API 호출 시 파라미터 누락
- **문제**: API 테스트 호출 시 API ID 파라미터가 전달되지 않음
- **영향도**: 높음 - API 테스트 기능 불가
- **근본 원인**: 프론트엔드에서 파라미터 검증 및 전달 로직 부재

### 5. UI 기본 기능 제한
- **문제**: TextInput 영역에서 텍스트 드래그 및 Ctrl+A 전체 선택 불가
- **영향도**: 중간 - 사용자 편의성 저하
- **근본 원인**: CSS `user-select` 속성 설정 및 이벤트 핸들러 문제
